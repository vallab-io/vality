## Vality SaaS Demo Video 시나리오 (75–90초, 미니멀 톤)

- **설명(가이드/구성)**: 한글
- **멘트/나레이션(VO)**: 영어
- **컨셉**: 과하게 화려하지 않게, 실제 제품 화면 중심(95%+), 짧은 텍스트 오버레이만 사용
- **최종 산출물**: “촬영/편집자가 그대로 따라하면 완성되는” 타임코드 스크립트 + 샷리스트 + 캡션 팩

---

## 0) 크리에이티브 방향 (Tone & Rules)

- **톤**: 차분하고 자신감 있게, “제품이 말하게” 만들기
- **템포**: 빠르되 정신없지 않게 (UI 1.15–1.25x 정도만)
- **연출**: 줌/하이라이트 최소(2–3회), 과한 모션그래픽 금지
- **사운드**: 미니멀 비트 + 클릭/우쉬 SFX 약하게
- **자막**: 무음 시청 대비해서 전 구간 캡션 권장(문장 짧게)
- **텍스트 원칙(중요)**:
  - 별도 **온스크린 카피(오버레이 문구)**는 최대한 쓰지 않음
  - 대신 **VO(나레이션) 멘트를 그대로 자막으로** 노출해서 미니멀 톤 유지
  - UI 이해가 꼭 필요한 경우에만 **아주 작은 UI 하이라이트(원/밑줄/줌 1~2회)**로 보조
- **편집 원칙**:
  - 설명을 길게 하지 말고 “보여주면 이해되는 흐름”으로 구성
  - 전환은 **하드컷** 위주, 섹션 전환에만 약한 whoosh(6–8프레임)
  - 화면 오버레이는 **짧고 적게(6–10개)**만

## 1) 영상 목표 (한 문장)

- 한 번 발행하면 **이메일 발송 + 웹(SEO 아카이브) + 구독 관리**까지 자연스럽게 이어지는 흐름을 보여준다.

---

## 2) 데모 촬영 준비(중요) — “영상이 깔끔해지는 조건”

> 아래 준비가 되어 있으면 촬영 때 망설이는 장면(버벅임, 로딩 대기, 수정/삭제)을 크게 줄일 수 있음

- **데모용 계정/뉴스레터**
  - 데모용 뉴스레터 1개 생성해두기 (예: `Vality Weekly`)
  - 프로필 이미지/뉴스레터 커버는 심플한 이미지 1장으로 통일
- **데모용 이슈 텍스트(촬영 중 실제로 입력할 문구)**
  - **Title**: `3 Tiny Habits That Make Writing Easier`
  - **Slug**: `tiny-writing-habits`
  - **Description**: `A short issue about building a writing routine — without burning out.`
  - **Body (짧은 본문 샘플)**:
    - Heading: `Start smaller than you think`
    - Bullets:
      - `Write for 10 minutes. Stop on purpose.`
      - `Capture ideas daily. Publish weekly.`
      - `Edit later. Publish sooner.`
    - Quote: `Consistency beats intensity.`
  - **Image 1장**: 1200×630 또는 1600×900 정도 (너무 화려하지 않은 사진/일러스트)
- **구독자 상태 데모**
  - 테스트 이메일 2~3개(Active/Pending/Unsubscribed) 만들어두면 “리스트가 빈 화면”이 되지 않음
- **공개 페이지 URL**
  - 공개 페이지가 실제로 뜨는 상태(SEO 아카이브)를 보여줘야 하므로, 발행 후 바로 열리는 URL 동작 확인
- **UI 표시 언어**
  - 영상 타겟이 영어라면 UI도 영어로 통일(가능하면). 아니면 화면 텍스트/캡션으로 영어 보정.

## 3) 타임라인 스크립트 (Timecode / 화면 / VO / 온스크린)

> 형식: **Time / 화면(샷) / VO(EN) / 자막(VO 동일)**  
> 별도 온스크린 오버레이 문구는 사용하지 않고, **VO 문장을 그대로 자막으로** 보여주는 방식.

### 0:00–0:06 — Hook (문제 → 약속)
- **화면(샷)**:
  - 블러 처리된 에디터 화면 위로 타이틀 카드(살짝 움직임)
  - 마우스 커서는 가만히(불필요한 움직임 금지)
- **VO (EN)**:
  - “Writing is hard enough. Publishing shouldn’t be.”
- **자막(EN)**:
  - VO와 동일 (한 줄씩 짧게 끊기)

### 0:06–0:14 — 대시보드(맥락 최소)
- **화면(샷)**:
  - `Dashboard` → `Vality Weekly`가 보이는 화면(1초 정도)
  - `Issues` 탭 클릭 
  - “New issue” 버튼을 시선이 가는 위치로 두기(커서/줌 최소)
- **VO (EN)**:
  - “Meet Vality—built for creators who want to publish consistently.”
- **자막(EN)**:
  - VO와 동일

### 0:14–0:26 — 이슈 생성 → 에디터 진입(첫 ‘아하’)
- **화면(샷)**:
  - “New issue” 클릭 → 에디터 진입
  - 타이핑 시작 전 0.5초 정도 정지(시청자가 UI를 인지)
- **VO (EN)**:
  - “Start where it matters—your words.”
- **자막(EN)**:
  - VO와 동일

### 0:26–0:44 — 작성/편집(핵심 기능 2~3개만)
- **화면(샷)**:
  - Title 입력: `3 Tiny Habits That Make Writing Easier`
  - 본문에 헤딩/불릿/인용구를 순서대로 추가
  - 이미지 업로드 1회
  - (가능하면) Preview 미리보기 1초 보여주기(이메일, 아카이브)
- **VO (EN)**:
  - “Draft in a clean editor, add images, and keep it simple.”
- **자막(EN)**:
  - VO와 동일

### 0:44–1:02 — Publish 설정(타이틀/슬러그/설명)
- **화면(샷)**:
  - “Publish” 클릭 → Publish 모달/다이얼로그 오픈
  - Slug 입력: `tiny-writing-habits`
  - Description 입력: `A short issue about building a writing routine — without burning out.`
  - Slug 중복 체크가 “Available”처럼 보이게(또는 오류가 안 나게)
  - Preview 영역 1초 보여주기

  - “Set your slug and a short description. You’re ready to publish.”
- **자막(EN)**:
  - VO와 동일
- **UI 하이라이트(텍스트 없이)**:
  - (선택) Slug 입력칸에 짧은 하이라이트, “Available” 뱃지/상태만 화면에 자연스럽게 보이게

### 1:02–1:14 — 발행 → 이메일로 먼저 확인
- **화면(샷)**:
  - Publish 확정 클릭(이 순간은 **속도 정상**)

### 1:14–1:26 — 웹 아카이브(공개 페이지) 확인

## 4) 자막(VO) 스타일 가이드

- **원칙**: VO 문장을 그대로 자막으로 사용 (별도 온스크린 문구 없음)
- **표기**:
  - 한 줄 길이 짧게(가능하면 1줄, 길면 2줄)
  - 화면 하단 중앙 정렬
  - 색상은 흰색 + 얇은 그림자/반투명 배경(가독성 확보)
- **타이밍**:
  - 문장 시작 2–4프레임 먼저 등장 → 말 끝난 뒤 6–10프레임 후 퇴장
- **주의**:
  - 자막이 버튼/입력칸을 가리지 않도록, 필요한 경우 자막 위치를 약간 올리기

## 5) 촬영 샷리스트 (레코딩 체크리스트)

- **A1** Dashboard에서 뉴스레터 카드/리스트가 보이는 장면(1초)
- **A2** New issue 클릭 → 에디터 진입(1초)
- **A3** Editor: 제목 입력 → 본문(헤딩/불릿/인용) → 이미지 업로드 → (선택) preview 토글
- **A4** Publish 모달: Slug/Description 입력 + “Available” 상태 + preview 영역
- **A5** Publish 완료(토스트/상태)
- **A6** Public 웹 페이지: 첫 화면(타이포/여백) + subscribe 위치로 스크롤
- **A7** Subscribe 입력/성공 상태
- **A8** Subscribers 리스트(상태/검색)
- **A9** Analytics 오버뷰(핵심 지표)
- **A10** End card

## 6) 촬영/편집 가이드 (미니멀 톤 유지)

- **해상도**: 1440p 이상(줌 대비), 가능하면 4K
- **마우스 커서**: 큰 커서 or 클릭 하이라이트(은은하게)
- **속도**: 대부분 1.15–1.25x, Publish 클릭 순간은 정상 속도로 잠깐
- **줌**: 2–3회만
  - Publish 모달
  - Subscribe
  - Analytics
- **전환**: 하드컷 위주 + 섹션 전환에만 약한 whoosh(6–8프레임)
- **음악**: 미니멀(80–100BPM), -18~-14 LUFS 정도
- **캡션**: 항상 on (짧게 끊기)
- **주의(자주 망치는 포인트)**:
  - 커서가 “계속 돌아다니는” 장면 금지(불안해 보임)
  - 한 화면에 너무 많은 기능을 넣지 않기(핵심 2~3개만)
  - 로딩 대기/오타 수정 장면은 컷 편집으로 제거
